Please provide a valid XYZ tile layer URL.

An XYZ tile layer URL can have many different formats, such as:
  • https://osm.gs.mil/tiles/default/{z}/{x}/{y}.png
  • https://osm.gs.mil/tiles/default_pc/{z}/{x}/{y}.png
  • https://osm.gs.mil/tiles/humanitarian/{z}/{x}/{y}.png
  • https://osm.gs.mil/tiles/default/{z}/{x}/{y}.png
  • https://services.arcgisonline.com/ArcGIS/rest/services/
    World_Street_Map/MapServer/tile/{z}/{y}/{x}
Keep in mind that the positions of {x}, {y}, and {z} can vary between URLs.

Although the {x}, {y}, and {z} portions of each URL can be replaced with an integer value, it is not recommended to do so as this will only provide a single tile of the layer.

OpenSphere currently supports two projections for XYZ tile layers: EPSG:3857 and EPSG:4326. If an XYZ tile layer does not appear as expected, consider reimporting it with a different projection.
